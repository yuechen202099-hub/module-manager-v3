# V2.1 Local Test Build

## Goal

V2.1 local test build proves the core business workflow before cloud investment:

- Read `AGENTS.md` before changing this build or its documentation.
- Import the three provided Excel workbooks.
- Match stage rows to total catalog rows with the V1.3 barcode rule.
- Use total catalog as the only installation address source.
- Claim and release a review task.
- Split tasks by terminal.
- Block task claiming when a terminal has no scan records.
- Store scan/photo URL references only after scan import.
- Load photos from URL during review without downloading to local disk.
- Classify URL-loaded photos during review.
- Review material groups locally.
- Save group status and notes in the in-memory simulation.
- Show live progress statistics.

## V2.1 Task And Photo Rules

- Task hall is grouped by terminal.
- A terminal task can be claimed only when `scan_rows > 0`.
- Terminals without scan records stay visible for management visibility, but their claim button is disabled.
- Scan rows with images are treated as URL photo references in local simulation.
- URL photo references start as `unclassified`.
- Photos must be loaded from their URL when viewed. Do not download or cache the image files to local disk.
- Reviewers must be able to classify photos primarily through keyboard shortcuts as:
  - `before_box`
  - `after_box`
  - `module_meter`
  - `collector_barcode`
  - `other`
- Missing or unavailable URL photos cannot be classified.
