# Database Model Boundaries

## Required Tables

- `users`
- `roles`
- `user_roles`
- `projects`
- `total_catalog_rows`
- `stage_catalog_rows`
- `material_groups`
- `photos`
- `tasks`
- `task_groups`
- `review_records`
- `exceptions`
- `import_jobs`
- `export_jobs`
- `audit_logs`

## Required Indexes

- `total_catalog_rows(project_id, meter_match_key)`
- `stage_catalog_rows(project_id, meter_match_key)`
- `material_groups(project_id, meter_match_key)`
- `material_groups(task_id, status)`
- `photos(group_id, sha256)`
- `tasks(project_id, status)`
- `task_groups(task_id, group_id)`

## Integrity Rules

- Total catalog address must be stored per row and must not be deduplicated.
- `meter_match_key` is generated by backend code and persisted for query performance.
- `photos.sha256` prevents duplicate photo records in the same group.
- Task claim and release must be transactional.
