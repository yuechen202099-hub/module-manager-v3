## User Issue

- 管理员后台中资料组明明已有模块号，却提示“缺少模块资产编号”。
- 驾驶舱“图片准确率”显示 `0%`，应检照片全部落为“无法识别”。

## Root Cause

- 归档校验只读取照片字段 `asset_no`，没有读取资料组级 `module_asset_no`、`construction_module_asset_no` 或照片级 `module_asset_no`，导致展示口径和校验口径不一致。
- 生产照片存储在 OSS，条码复核拿到的是 `oss://bucket/key`，原扫码器没有通过服务端签名 URL 读取图片。
- `zxing-cpp==3.0.0` 不能直接读取文件路径字符串；照片需要先用 Pillow 解码为图像对象后再传入 `read_barcodes`。

## Changed Scope

- 统一资料组归档校验的模块号提取口径。
- OSS 照片条码复核改为使用服务端签名 URL 下载原图。
- 条码复核统一用 Pillow 解码图片后调用 zxing-cpp。
- 新增回归测试覆盖模块号别名、重复模块号别名、本地图片扫码、OSS 图片扫码。
- 更新中文版本说明到 `V3.0.38`。
