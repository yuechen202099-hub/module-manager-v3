## 问题描述

V3.0.56 已将账号转换为姓名，但终端明细安装人员聚合仍只读取资料组 `raw_data` 中的 `installer`、`constructor`、`creator` 字段。生产库只 13 个已上传资料组存在这些字段；但审阅资料卡展示的安装人员来自照片 `creator`，1876 个已上传资料组均有照片创建人，因此终端明细聚合漏算了绝大多数资料组。

## 修复内容

- PostgreSQL 任务列表 `installer_distribution` 优先按资料组内有效照片 `Photo.creator` 聚合。
- 每个资料组内同一安装人员只计一次，避免同一组 4 张照片重复计数。
- 如果同一资料组由多名安装人员上传照片，则分别计入这些安装人员的完成占比。
- 只有当资料组没有有效照片创建人时，才回退到资料组 `raw_data` 的 `installer`、`constructor`、`creator` 字段。
- JSON 本地状态路径同步按照片 `creator` 优先聚合，保持开发验证和生产口径一致。
- 账号到姓名映射继续沿用 V3.0.56 的规则。
