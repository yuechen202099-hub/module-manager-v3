## 问题描述

项目驾驶舱顶部统计卡仍需要等待 `/local-test/summary` 实时聚合资料组、照片准确率和资料组条码准确率。即使 V3.0.59 已把接口改为轻量 SQL 聚合，用户每次进入页面仍会触发一次服务端重算，弱服务器和移动端网络下仍会看到明显加载等待。

## 修复内容

- 新增项目驾驶舱统计服务器快照缓存，默认每 5 分钟在后台生成一次 `/local-test/summary` 快照。
- 快照同时保存在服务器内存和 `data/project-board-cache` JSON 文件中，服务重启后可先读取上次快照。
- `/local-test/summary` 默认优先返回缓存快照；缓存过期时先返回旧快照，再后台异步刷新，避免用户进入页面时阻塞等待。
- 管理员点击驾驶舱刷新按钮时会带 `refresh=true`，可强制重新生成统计快照。
- 不新增数据库表，不修改生产业务数据，不触发条码扫描或条码重算。
