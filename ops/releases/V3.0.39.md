## User Issue

- 项目看板“异常与缺照/待派发”列表中仍出现“缺少模块资产编号”。
- 用户截图中的资料组已经有模块号，继续显示该异常会误导管理员和施工派发。

## Root Cause

- V3.0.38 修复了归档校验函数的模块号识别口径，但 PostgreSQL 的异常列表入口仍直接读取历史 `exception_note`。
- 生产库里部分历史记录 `status=rejected`、`exception_note=缺少模块资产编号`，但 `has_archive_blocker=false` 且模块号已存在。
- 前端 `exceptionReasonText()` 在 `exception_reasons` 为空时会回退展示 `exception_note`，因此待派发列表继续显示旧自动异常。

## Changed Scope

- PostgreSQL 异常列表读取前，会用数据库资料组字段和活跃照片字段重新校验归档异常。
- 新增 PostgreSQL 专用模块号索引，避免使用 JSON state 的模块号映射。
- 自动生成的过期归档异常备注会落库清理。
- 没有真实异常的历史 rejected 资料组会恢复为待审状态。
- 真实异常继续保留：缺少采集器信息、资料组照片不足、模块号重复。
