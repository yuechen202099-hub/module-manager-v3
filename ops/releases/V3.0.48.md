# V3.0.48 生产发布记录

## 用户需求

- 条码后台维护每天 00:00 自动筛选未分析过的资料组，按 50 个一批处理。
- 没运行过的批次要继续补跑。
- 图片已分类且条形码扫描通过的资料组自动归档。
- 有问题的资料组保留人工判断。

## 优先级

P0 生产维护修复。

## 变更摘要

- 修复 `archive` 模式只检查前 50 个资料组的问题。
- 维护脚本现在解析每批 `groups_seen`，归档模式会按 `--group-offset` 每次推进 50 组，直到没有更多候选窗口。
- 保留失败即停：命令失败、JSON 报告异常、或 `errors > 0` 都会立即停止并写日志。
- 保留服务器保护：每批 50 组，批间暂停，不做全量一次性扫描。
