# V3.0.71 Production Release Record

## Change Type

- Bug fix, `+0.01`
- Scope: manual barcode confirmation status, project-board barcode accuracy cache invalidation, and barcode review list filtering/search.
- Version: `3.0.71`
- Date: 2026-06-30

## Change Summary

- Manual confirmation now moves a material group out of unreadable/mismatched barcode review lists and into the passed list.
- Project-board barcode accuracy is recalculated after manual barcode confirmation, photo rescans, and photo category saves by invalidating the cached project-board summary.
- Review workbench photo labels treat manually confirmed groups as `扫码通过` instead of showing stale per-photo abnormal states.
- Barcode review dialog now supports `通过` and `全部` filters.
- Barcode review dialog now supports searching within the active filter by meter number, module number, collector number, terminal, installer, and related detected values.

## Safety Notes

- No database migration is included.
- No production `.env`, `data`, `uploads`, OSS objects, or PostgreSQL data are modified by the release package.
- Existing raw photo barcode evidence is preserved; manual confirmation remains group-level evidence with audit logging.
- Frontend barcode review still reads server-side results only and does not perform foreground scanning.
