## 问题描述

任务领取页在审阅进度或施工进度达到 100% 后，虽然状态标签已经显示完成，但主操作按钮仍可能显示“释放工单”和“改派施工”，管理员和审阅员仍能看到可操作按钮，容易误以为已完成终端还需要继续处理或可以继续改派。

## 修复内容

- 审阅完成后：任务主按钮显示“已审阅”，并进入禁用状态，不再显示“释放工单”。
- 施工完成后：管理员施工按钮显示“已施工”，并进入禁用状态，不再显示“指派施工/改派施工”。
- 更多菜单：已审阅终端禁用释放动作，已施工终端禁用指派/改派动作。
- 操作防御：领取、释放、指派弹窗和更多菜单 command 处理层同步拦截完成态，避免旧 DOM 状态或程序化触发绕过按钮禁用。
- 验证门禁：扩展 `scripts/verify_claim_tasks_completion_status.js`，覆盖完成态按钮文案和禁用规则。
- 发布包门禁：`scripts/verify-client-release.py` 将完成态按钮门禁脚本列为必备文件。
