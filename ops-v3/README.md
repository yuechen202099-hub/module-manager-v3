# V3 轻量运维记录

本目录不再使用旧的多 Agent 强流程。当前协作规则以以下文件为准：

- `AGENTS.md`
- `docs/WORK_RULES.md`

## 目录用途

- `tasks/`：按版本保存代码改动任务记录。
- `releases/`：保存发布记录。
- `acceptance/`：保存重要验收记录。
- `db-changes/`：保存数据库迁移、数据修复、dry-run、回滚记录。
- `incidents/`：保存生产事故和阻塞问题记录。

## 当前任务记录路径

```text
ops-v3/tasks/V3.0.2/
```

所有代码改动都要建立简短任务记录。文档和协作规则改动不强制建立任务记录。

## 生产底线

- 不覆盖生产 `.env`。
- 不覆盖生产 `data`。
- 不覆盖生产 `uploads`。
- 不直接修改 OSS 生产对象。
- 不直接修改 PostgreSQL 生产数据。
- 数据库迁移、数据修复、生产数据相关操作必须单独确认。
